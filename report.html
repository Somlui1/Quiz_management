1. ภาพรวมระบบ (System Overview)

Quizmaster Pro คือระบบจัดสอบออนไลน์แบบ Monolithic ที่ถูกออกแบบมาเพื่อใช้ภายในองค์กร (Enterprise) สำหรับจัดการการสอบ/ประเมินความรู้พนักงาน โดยรองรับการทำงานแบบ Real-time ผ่าน Server-Sent Events (SSE)

Quizmaster Pro is a monolithic online examination system designed for internal enterprise use. It manages employee knowledge assessments and supports real-time monitoring via Server-Sent Events (SSE).

Tech Stack

Layer Technology Purpose
Frontend React 19 + TypeScript Single-Page Application (SPA)
Styling TailwindCSS v4 Utility-first CSS framework
Backend Node.js + Express 4 REST API Server
Database SQLite via better-sqlite3 Embedded relational database
Auth jsonwebtoken + bcryptjs JWT tokens & password hashing
Build Vite + esbuild Frontend bundling & server compilation
Deployment Docker + NGINX + ngrok Containerized with reverse proxy

3. สถาปัตยกรรมฐานข้อมูล (Database Architecture)

ระบบใช้แนวคิด Dual-Database Architecture โดยแยกฐานข้อมูลหลัก (Master) ออกจากฐานข้อมูลห้องสอบ (Campaign) เพื่อประสิทธิภาพและความเป็นอิสระ

The system uses a Dual-Database Architecture, separating the master database from per-campaign databases for performance and isolation.

5. การไหลของข้อมูล (Data Flow)

5.1 Admin Login Flow (การเข้าสู่ระบบผู้ดูแล)

sequenceDiagram
    participant Admin as Admin Browser
    participant NGINX as NGINX Proxy
    participant Server as Express Server
    participant MasterDB as master.db
    participant ESS as ESS Aapico API

    Admin->>NGINX: POST /api/auth {identifier, password, isAdminLogin: true}
    NGINX->>NGINX: Rate limit check (auth_limit: 5r/s)
    NGINX->>Server: Forward request

    alt System Account
        Server->>MasterDB: SELECT * FROM admins WHERE type='system'
        MasterDB-->>Server: Admin row
        Server->>Server: bcrypt.compareSync(password, hash)
        Server->>Server: jwt.sign({isAdmin: true, role, type: 'system'})
        Server-->>Admin: {jwt: "eyJ...", user: {...}}
    else ESS Account
        Server->>ESS: POST https://ess.aapico.com/auth/local
        ESS-->>Server: {jwt, user: {idCard, birthday}}
        Server->>ESS: GET /employees/preinfo?id_card=...
        ESS-->>Server: Employee profile
        Server->>MasterDB: SELECT * FROM admins WHERE type='ess' AND id=...
        MasterDB-->>Server: ESS Admin row (role check)
        Server->>Server: jwt.sign({isAdmin: true, role, type: 'ess'})
        Server-->>Admin: {jwt: "eyJ...", user: {...}}
    end

    Admin->>Admin: localStorage.setItem("authenticated_admin_jwt", jwt)

5.2 Student Login & Exam Flow (การเข้าสอบของนักเรียน/พนักงาน)

sequenceDiagram
    participant Student as Student Browser
    participant NGINX as NGINX Proxy
    participant Server as Express Server
    participant MasterDB as master.db
    participant CampDB as campaign_{id}.db
    participant ESS as ESS Aapico API
    participant LiveBoard as Admin Live Board (SSE)

    Note over Student: Step 1: Login via ESS
    Student->>NGINX: POST /api/auth {identifier, password}
    NGINX->>Server: Forward
    Server->>ESS: POST /auth/local → GET /employees/preinfo
    ESS-->>Server: Employee profile
    Server->>Server: jwt.sign({employeeId, name, type: "student"})
    Server-->>Student: {jwt: "eyJ...", user: {...}}
    Student->>Student: Store JWT in userProfile.jwt

    Note over Student: Step 2: Check Remaining Attempts
    Student->>NGINX: GET /api/campaigns/{id}/attempts/{empId} [Bearer JWT]
    NGINX->>Server: Forward
    Server->>Server: authenticateStudentJWT → verify identity
    Server->>CampDB: SELECT COUNT(*) FROM submissions WHERE user_identifier=?
    CampDB-->>Server: {count: N}
    Server-->>Student: {allowed: true/false, attemptsCount: N}

    Note over Student: Step 3: Start Exam Attempt
    Student->>NGINX: POST /api/campaigns/{id}/start-attempt [Bearer JWT]
    NGINX->>Server: Forward
    Server->>Server: authenticateStudentJWT → verify identity match
    Server->>MasterDB: SELECT * FROM campaigns (get questions)
    Server->>Server: Randomize & shuffle questions/choices
    Server->>CampDB: INSERT INTO attempts (store randomized set with answers)
    Server-->>Student: {attemptId, questions: [...]} (answers STRIPPED)

    Note over Student: Step 4: Join Live Lobby
    Student->>NGINX: POST /api/campaigns/{id}/join [Bearer JWT]
    NGINX->>Server: Forward
    Server->>Server: authenticateStudentJWT → verify identity match
    Server->>Server: activeParticipants.set(empId, {...})
    Server->>LiveBoard: SSE push {type: "join", participant: {...}}

    Note over Student: Step 5: Submit Answers
    Student->>NGINX: POST /api/campaigns/{id}/submit [Bearer JWT]
    NGINX->>Server: Forward
    Server->>Server: authenticateStudentJWT → verify identity match
    Server->>CampDB: SELECT questions_json FROM attempts WHERE id=?
    Server->>Server: Grade answers server-side (compare with correctIndex)
    Server->>CampDB: INSERT INTO submissions
    Server->>LiveBoard: SSE push {type: "submission", submission: {...}}
    Server-->>Student: {scorePercent, passed, answersEvaluation: [...]}

5.3 Real-Time Live Board (SSE) (กระดานติดตามสด)

sequenceDiagram
    participant Admin as Admin Browser
    participant Server as Express Server

    Admin->>Server: GET /api/campaigns/{id}/live (SSE Connection)
    Server->>Server: res.setHeader("Content-Type", "text/event-stream")
    Server->>Server: liveClients.get(id).push(res)
    Server-->>Admin: data: {"type": "connected"}

    loop Real-Time Events
        Note over Server: When student joins
        Server-->>Admin: data: {"type": "join", "participant": {...}}

        Note over Server: When student submits
        Server-->>Admin: data: {"type": "submission", "submission": {...}}

        Note over Server: When admin resets campaign
        Server-->>Admin: data: {"type": "reset"}
    end

    Admin->>Server: Connection closed (onclose)
    Server->>Server: Remove from liveClients array

JWT Token Types

Token Type Signed By Payload Used By
Admin JWT JWT_SECRET {username_or_id, role, type, isAdmin: true} Admin panel APIs
Student JWT JWT_SECRET {employeeId, name, type: "student"} Student exam APIs

Token Lifecycle

Event Storage Key Action
Admin Login localStorage["authenticated_admin_jwt"] Set token
Admin Logout localStorage["authenticated_admin_jwt"] Remove token
Student Login userProfile.jwt (React state) Set token
Student Logout localStorage["authenticated_user_profile"] Remove profile + token
